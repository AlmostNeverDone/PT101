# Network Reconnaissance and Active Scanning with Nmap and Wireshark
使用 Nmap 與 Wireshark 進行網路偵察與主動式掃描

---------

<h2>Outline 簡介</h2>

This project demonstrates how active network scanning can be used to discover hosts, identify open ports, and analyze network communications within a controlled penetration-testing environment.

本專題示範如何在受控的滲透測試環境中，透過主動式網路掃描來發現主機、識別開放埠號並分析網路通訊。

Using Nmap, we perform reconnaissance scans to enumerate services, detect operating systems, and identify potential attack surfaces on a target system. These scans actively interact with the target by sending network packets and analyzing the responses returned by the host. 

透過 Nmap，我們執行偵察掃描以列舉服務、辨識作業系統並找出目標系統可能的攻擊面。此類掃描會主動向目標主機發送封包並分析回應內容。 

To further understand the network behavior during the scanning process, Wireshark is used to capture and analyze packet-level traffic generated by the scans. This enables deeper visibility into network protocols, request patterns, and system responses.

為了進一步理解掃描過程中的網路行為，我們使用 Wireshark 擷取並分析掃描所產生的封包流量，從而深入觀察網路協定、請求模式與系統回應。

Overall, this project highlights how scanning tools and packet analysis can be combined to better understand network exposure and system behavior during security assessments.

總體而言，本專題展示如何結合掃描工具與封包分析，以更全面地理解系統在安全評估過程中的網路暴露面與行為。

<h2>Key Learning Outcomes 主要學習成果</h2>


* Understand the concept of active scanning and how it differs from passive network reconnaissance.<br/>
  理解主動式掃描的概念以及其與被動式網路偵察的差異。
  
* Learn how to use Nmap to discover hosts, identify open ports, and enumerate running services.
  學習使用 Nmap 發現主機、辨識開放連接埠並列舉運行中的服務。
  
* Perform different scanning techniques such as basic scans, aggressive scans, and host discovery scans.
  實作不同掃描方式，例如 基本掃描、進階掃描與主機探測掃描。
  
* Analyze scan results to identify potential security exposure and network services.
  分析掃描結果以識別可能的 安全暴露面與網路服務。

* Capture and analyze network packets generated during scans using Wireshark.
  使用 Wireshark 擷取並分析掃描過程中產生的網路封包。

* Develop foundational skills for penetration testing, reconnaissance, and network traffic monitoring.
  培養滲透測試、偵察和網路流量監控的基礎技能。

<h2>Materials and Methods 材料與方法</h2>

[Environment]
* Hyper-V virtualization platform (虛擬化平台)</b>
* Kali Linux (attacker machine)</b>
* Metasploitable vulnerable VM (target system)</b>
* Nmap network scanner (網路掃描器)</b>
* Wireshark packet analyzer (封包分析器)</b>

[Tasks]
* Perform a Basic Nmap Scan on a Public Target (對公開目標執行基本 Nmap 掃描)</b>
* Perform an Aggressive Nmap Scan for Service Enumeration (執行進階 Nmap 掃描以列舉服務)</b>
* Launch a Target VM for Active Scanning (啟動目標虛擬機進行主動掃描)</b>
* Identify the Target System IP Address (確認目標系統 IP 位址)</b>
* Scan the Target Host for Open Ports and Services (掃描目標主機的開放埠與服務)</b>
* Perform Host Discovery Using Ping Scan (使用 Ping Scan 進行主機探測)</b>
* Launch Wireshark for Network Packet Capture (啟動 Wireshark 進行封包擷取)</b>
* Capture and Inspect Network Traffic Generated by Nmap (擷取並分析 Nmap 產生的網路流量)</b>

<h2>Results 專題結論</h2>

This project demonstrates how Wireshark can decrypt and analyze RDP traffic when the correct RSA private key is available. 

本專題示範了在取得正確的 RSA 私鑰後，如何使用 Wireshark 解密並分析 RDP 流量。

The captured session from Bob’s host was initially unreadable due to TLS encryption, but once the key was imported, Wireshark revealed clear RDP negotiation details, session metadata, and identifiers such as the mstshash=bucky cookie.

從 Bob 主機擷取的 RDP 會話因 TLS 加密而無法閱讀，但在匯入私鑰後，Wireshark 成功顯示 RDP 協商流程、會話中繼資料，以及 mstshash=bucky 等識別資訊。

Through filtering, protocol inspection, and TCP stream reconstruction, we confirmed the initiating host, validated the server endpoint, and verified the decrypted session’s integrity. This workflow highlights the importance of key recovery and structured analysis when handling encrypted network traffic.

透過過濾、協定檢視與 TCP stream 重建，我們確認了會話發起主機、驗證伺服器端點，並確保解密後的會話內容完整。此流程強調了在處理加密網路流量時，金鑰回收與結構化分析的重要性。

Overall, this project reinforces a key principle in network forensics: encrypted traffic is still analyzable when proper key material and methodology are applied.

總結而言，本專題強化了網路鑑識中的一項核心觀念：只要具備正確的金鑰與分析流程，加密流量依然能被有效解析與調查。
